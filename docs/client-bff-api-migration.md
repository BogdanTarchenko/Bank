# Client BFF API — Справочник для фронтенда и мобильных клиентов

## Общие принципы

- **Base URL:** `http://localhost:8084`
- **Авторизация:** каждый запрос требует заголовок `Authorization: Bearer <jwt_token>`
- **userId не передаётся клиентом** — BFF определяет пользователя из JWT автоматически
- **Владение ресурсами:** BFF проверяет, что счёт/кредит принадлежит текущему пользователю. Попытка обратиться к чужому ресурсу — `403`
- **Формат данных:** `Content-Type: application/json`

---

## 1. Счета (Accounts)

### Эндпоинты

#### `GET /api/v1/accounts` — Мои счета

Возвращает только счета текущего пользователя.

**Ответ** `200`:

```json
[
  {
    "id": 1,
    "userId": 10,
    "currency": "RUB",
    "balance": 15000.00,
    "accountType": "USER",
    "isClosed": false,
    "createdAt": "2026-03-20T10:00:00"
  }
]
```

---

#### `GET /api/v1/accounts/{id}` — Счёт по ID

**Ответ** `200` — если счёт принадлежит вам. `403` — если чужой.

---

#### `POST /api/v1/accounts` — Создать счёт

**Тело запроса:**

```json
{
  "currency": "RUB"
}
```

`userId` передавать **не нужно** — подставляется из JWT.

Допустимые значения `currency`: `RUB`, `USD`, `EUR`.

**Ответ** `201` — созданный счёт.

---

#### `DELETE /api/v1/accounts/{id}` — Закрыть счёт

**Ответ** `204` — успешно. `403` — чужой счёт. `409` — баланс не нулевой / счёт уже закрыт.

---

#### `POST /api/v1/accounts/{id}/deposit` — Пополнить

**Тело запроса:**

```json
{
  "amount": 1000.00
}
```

`amount` > 0.01.

**Ответ** `202` — принято. `403` — чужой счёт. `409` — счёт закрыт.

---

#### `POST /api/v1/accounts/{id}/withdraw` — Снять

**Тело запроса:**

```json
{
  "amount": 500.00
}
```

**Ответ** `202` — принято. `403` — чужой счёт. `422` — недостаточно средств. `409` — счёт закрыт.

---

#### `GET /api/v1/accounts/{id}/operations` — История операций

**Query-параметры:** `?page=0&size=20`

**Ответ** `200` — постраничный список:

```json
{
  "content": [
    {
      "id": 1,
      "accountId": 1,
      "type": "DEPOSIT",
      "amount": 1000.00,
      "currency": "RUB",
      "relatedAccountId": null,
      "exchangeRate": null,
      "description": "Пополнение",
      "createdAt": "2026-03-20T12:00:00"
    }
  ],
  "totalElements": 1,
  "totalPages": 1,
  "number": 0,
  "size": 20
}
```

Типы операций (`type`): `DEPOSIT`, `WITHDRAWAL`, `TRANSFER_IN`, `TRANSFER_OUT`.

## 2. Переводы (Transfers)

#### `POST /api/v1/transfers` — Перевести

**Тело запроса:**

```json
{
  "fromAccountId": 1,
  "toAccountId": 2,
  "amount": 100.00
}
```

- `fromAccountId` — должен принадлежать текущему пользователю, иначе `403`
- `toAccountId` — может быть любым (перевод другому пользователю допустим)
- `amount` > 0.01

**Ответ** `202` — принято. `403` — чужой счёт-источник. `422` — недостаточно средств. `409` — счёт закрыт.

---

## 3. Кредиты (Credits)

> **Новые эндпоинты.** Старый путь `/api/v1/proxy/credit/credits/...` заблокирован — возвращает `403`.

### Эндпоинты

#### `GET /api/v1/credits` — Мои кредиты

Возвращает только кредиты текущего пользователя.

**Ответ** `200`:

```json
[
  {
    "id": 1,
    "userId": 10,
    "accountId": 1,
    "tariffId": 1,
    "tariffName": "Стандарт",
    "principal": 50000.00,
    "remaining": 45000.00,
    "accruedInterest": 200.00,
    "interestRate": 0.15,
    "termDays": 365,
    "dailyPayment": 150.00,
    "status": "ACTIVE",
    "createdAt": "2026-03-01T10:00:00",
    "closedAt": null
  }
]
```

Статусы кредита (`status`): `ACTIVE`, `CLOSED`, `OVERDUE`.

---

#### `GET /api/v1/credits/{id}` — Кредит по ID

**Ответ** `200` — если ваш кредит. `403` — если чужой. `404` — не найден.

---

#### `POST /api/v1/credits` — Оформить кредит

**Тело запроса:**

```json
{
  "accountId": 1,
  "tariffId": 1,
  "amount": 50000.00,
  "termDays": 365
}
```

- `userId` передавать **не нужно** — подставляется из JWT
- `accountId` — должен принадлежать текущему пользователю, иначе `403`
- `termDays` — срок в **днях** (не месяцах), ограничен тарифом

**Ответ** `201` — созданный кредит. `400` — невалидная сумма или срок (вне диапазона тарифа). `403` — чужой счёт. `404` — тариф не найден.

---

#### `GET /api/v1/credits/{id}/payments` — График платежей

**Ответ** `200`:

```json
[
  {
    "id": 1,
    "creditId": 1,
    "amount": 150.00,
    "status": "PAID",
    "dueDate": "2026-03-02T00:00:00",
    "paidAt": "2026-03-02T09:15:00"
  }
]
```

Статусы платежа (`status`): `PENDING`, `PAID`, `OVERDUE`.

---

#### `POST /api/v1/credits/{id}/repay` — Погасить кредит

**Тело запроса:**

```json
{
  "amount": 5000.00
}
```

`amount` > 0.01.

**Ответ** `200` — обновлённый кредит. `403` — чужой кредит. `409` — кредит уже закрыт. `422` — недостаточно средств на счёте.

---

## 5. Обработка ошибок

Все ошибки возвращаются в **едином формате**:

```json
{
  "status": 422,
  "error": "Unprocessable Entity",
  "message": "Недостаточно средств на счёте 1. Запрошено: 3000.00, доступно: 2900.00",
  "timestamp": "2026-03-22T18:23:00"
}
```

### Каталог ошибок

| HTTP-код | Когда                                                        | Пример `message`                                                    |
|----------|--------------------------------------------------------------|----------------------------------------------------------------------|
| `400`    | Невалидные данные (срок/сумма кредита вне тарифа, пустые поля) | `"Срок кредита должен быть от 90 до 365 дней"`                    |
| `403`    | Попытка доступа к чужому ресурсу                             | `"Нет доступа к данному счёту"` / `"Нет доступа к данному кредиту"` |
| `404`    | Ресурс не найден                                             | `"Счёт не найден"` / `"Кредит не найден"` / `"Тариф не найден"`   |
| `409`    | Конфликт состояния                                           | `"Счёт закрыт"` / `"Баланс не нулевой"` / `"Кредит уже закрыт"`   |
| `422`    | Недостаточно средств                                         | `"Недостаточно средств на счёте 1. Запрошено: 3000.00, доступно: 2900.00"` |
| `500`    | Внутренняя ошибка сервера                                    | `"Внутренняя ошибка сервера"`                                      |
| `503`    | Сервис курсов валют недоступен                               | `"Ошибка при запросе курсов валют"`                                |

### Рекомендация

Обрабатывайте ошибки по `status`:
- `400` — показать `message` пользователю (ошибка ввода)
- `403` — показать сообщение об отказе в доступе
- `404` — ресурс не найден
- `409` — показать `message` (конфликт состояния)
- `422` — показать `message` (бизнес-ошибка)
- `500`/`503` — показать общее сообщение «Что-то пошло не так»

## 6. Миграция — что нужно изменить

### Обязательные изменения

| #  | Что сделать                                                                        |
|----|------------------------------------------------------------------------------------|
| 1  | Заменить `/api/v1/proxy/credit/credits` → `/api/v1/credits` во всех вызовах       |
| 2  | Убрать `userId` из тела запросов на создание счёта и кредита                       |
| 3  | В создании кредита: поле `termInMonths` → `termDays` (срок в днях, не в месяцах)  |
| 4  | Обрабатывать HTTP `403`, `409`, `422` (раньше приходили как `500`)                 |

### Ничего менять не нужно

| Компонент                                | Причина                          |
|------------------------------------------|----------------------------------|
| Счета `/api/v1/accounts/*`              | Путь и формат не изменились      |
| Переводы `/api/v1/transfers`            | Путь и формат не изменились      |
| Тарифы `/api/v1/proxy/credit/tariffs`   | Работают как раньше              |
| Рейтинг `/api/v1/proxy/credit/users/*/credit-rating` | Работает как раньше |
| JWT-авторизация                          | Логика токенов не затронута      |
