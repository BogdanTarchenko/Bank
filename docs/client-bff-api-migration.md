# Client BFF API — Миграция для фронтенда и мобильных клиентов

## Что произошло

BFF-слой (`client-bff`) теперь проверяет **владение ресурсами** (счета, кредиты).  
Клиент не может получить доступ к чужим счетам и кредитам — BFF определяет пользователя из JWT автоматически.

---

## Base URL

```
http://localhost:8084
```

Все запросы требуют заголовок `Authorization: Bearer <jwt_token>`.

---

## 1. Счета (Accounts)

> **Путь не изменился.** Если вы уже используете `/api/v1/accounts` — ничего менять не нужно.

| Метод    | Эндпоинт                          | Тело запроса                                          | Ответ                       |
|----------|------------------------------------|-------------------------------------------------------|-----------------------------|
| `GET`    | `/api/v1/accounts`                | —                                                     | `200` — массив своих счетов |
| `GET`    | `/api/v1/accounts/{id}`           | —                                                     | `200` / `403`               |
| `POST`   | `/api/v1/accounts`                | `{ "currency": "RUB" }`                              | `201` — созданный счёт      |
| `DELETE` | `/api/v1/accounts/{id}`           | —                                                     | `204` / `403`               |
| `POST`   | `/api/v1/accounts/{id}/deposit`   | `{ "amount": 1000.00 }`                              | `202` / `403`               |
| `POST`   | `/api/v1/accounts/{id}/withdraw`  | `{ "amount": 500.00 }`                               | `202` / `403`               |
| `GET`    | `/api/v1/accounts/{id}/operations`| query: `?page=0&size=20`                             | `200` / `403`               |

### Важно

- **Не передавайте `userId`** в теле запроса — BFF берёт его из JWT.
- `GET /api/v1/accounts` без параметров — возвращает только ваши счета.
- При попытке обратиться к чужому счёту — `403`.

---

## 2. Переводы (Transfers)

> **Путь не изменился.**

| Метод  | Эндпоинт             | Тело запроса                                                        | Ответ         |
|--------|-----------------------|----------------------------------------------------------------------|---------------|
| `POST` | `/api/v1/transfers`  | `{ "fromAccountId": 1, "toAccountId": 2, "amount": 100.00 }`      | `202` / `403` |

### Важно

- BFF проверяет, что `fromAccountId` принадлежит текущему пользователю.
- `toAccountId` может быть любым (перевод другому пользователю допустим).

---

## 3. Кредиты (Credits) — НОВЫЕ ЭНДПОИНТЫ

> **Старый путь** `/api/v1/proxy/credit/credits/...` **больше не работает** — возвращает `403`.  
> **Новый путь:** `/api/v1/credits/...`

| Метод  | Эндпоинт                        | Тело запроса                                                                              | Ответ                         |
|--------|----------------------------------|--------------------------------------------------------------------------------------------|-------------------------------|
| `GET`  | `/api/v1/credits`               | —                                                                                          | `200` — массив своих кредитов |
| `GET`  | `/api/v1/credits/{id}`          | —                                                                                          | `200` / `403`                 |
| `POST` | `/api/v1/credits`               | `{ "accountId": 1, "tariffId": 1, "amount": 50000.00, "termInMonths": 12 }`              | `201` / `403`                 |
| `GET`  | `/api/v1/credits/{id}/payments` | —                                                                                          | `200` / `403`                 |
| `POST` | `/api/v1/credits/{id}/repay`    | `{ "accountId": 1, "amount": 5000.00 }`                                                  | `200` / `403`                 |

### Что изменить в клиенте

1. Заменить все вызовы `/api/v1/proxy/credit/credits` → `/api/v1/credits`
2. Убрать `userId` из тела запросов — BFF подставляет автоматически
3. При создании кредита (`POST /api/v1/credits`) — `accountId` должен принадлежать текущему пользователю, иначе `403`

## 4. Справочники — без изменений

Эти эндпоинты работают через прокси и НЕ требуют владения:

| Метод | Эндпоинт                                       | Описание         |
|-------|-------------------------------------------------|------------------|
| `GET` | `/api/v1/proxy/credit/tariffs`                 | Список тарифов   |
| `GET` | `/api/v1/proxy/credit/users/{id}/credit-rating`| Кредитный рейтинг|

---

## 5. Обработка ошибок

При отказе в доступе BFF возвращает **единый формат**:

```
HTTP/1.1 403 Forbidden
Content-Type: application/json
```

```json
{
  "status": 403,
  "error": "Forbidden",
  "message": "Нет доступа к данному счёту",
  "timestamp": "2026-03-22T15:30:00"
}
```

Возможные значения `message`:
- `"Нет доступа к данному счёту"` — для операций со счетами
- `"Нет доступа к данному кредиту"` — для операций с кредитами

### Рекомендация для клиента

Обрабатывайте HTTP `403` глобально. Тело ответа всегда содержит `message` с человекочитаемым текстом, который можно показать пользователю.

## 6. Сводка: что нужно изменить

### Минимальные изменения (обязательно)

| # | Действие                                                                 |
|---|--------------------------------------------------------------------------|
| 1 | Заменить `/api/v1/proxy/credit/credits` → `/api/v1/credits` во всех вызовах |
| 2 | Убрать `userId` из тела запросов на создание счёта и кредита              |
| 3 | Добавить обработку HTTP `403` (если ещё нет)                             |

### Ничего менять не нужно

| Компонент            | Причина                                      |
|----------------------|----------------------------------------------|
| Счета (`/api/v1/accounts`) | Путь и формат не изменились             |
| Переводы (`/api/v1/transfers`) | Путь и формат не изменились          |
| Тарифы (`/api/v1/proxy/credit/tariffs`) | Работают как раньше           |
| JWT-авторизация      | Логика токенов не затронута                  |
